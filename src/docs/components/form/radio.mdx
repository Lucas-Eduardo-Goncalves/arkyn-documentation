O componente `Radio` é composto por dois elementos principais: `RadioGroup` e `RadioBox`. O `RadioGroup` gerencia o estado e contexto dos botões de rádio, enquanto o `RadioBox` representa cada opção individual. É ideal para seleções exclusivas onde apenas uma opção pode ser escolhida por vez.

## Importação

```tsx
import { RadioGroup, RadioBox } from "@arkyn/components";
```

## Uso Básico

O uso básico requer um `RadioGroup` contendo um ou mais `RadioBox` com valores únicos.

<div className="noContent column">
  <RadioGroup name="basic-radio" label="Escolha uma opção">
    <RadioBox value="option1">Opção 1</RadioBox>
    <RadioBox value="option2">Opção 2</RadioBox>
    <RadioBox value="option3">Opção 3</RadioBox>
  </RadioGroup>
</div>

## Callback onChange

O `RadioGroup` oferece um callback que é chamado sempre que a seleção muda.

```tsx
const [selectedValue, setSelectedValue] = useState("");

const handleRadioChange = (value) => {
  console.log("Valor selecionado:", value);
  setSelectedValue(value);

  // Lógica adicional baseada no valor
  if (value === "premium") {
    // Habilitar recursos premium
  }
};

<RadioGroup
  name="subscription"
  label="Plano de assinatura"
  onChange={handleRadioChange}
  value={selectedValue}
>
  <RadioBox value="basic">Básico</RadioBox>
  <RadioBox value="premium">Premium</RadioBox>
  <RadioBox value="enterprise">Enterprise</RadioBox>
</RadioGroup>;
```

## Radio Controlado vs Não Controlado

### Não controlado (defaultValue)

```tsx
// O componente gerencia seu próprio estado
<RadioGroup name="theme" defaultValue="light">
  <RadioBox value="light">Claro</RadioBox>
  <RadioBox value="dark">Escuro</RadioBox>
</RadioGroup>
```

### Controlado (value + onChange)

```tsx
const [theme, setTheme] = useState("light");

// O estado é gerenciado externamente
<RadioGroup name="theme" value={theme} onChange={setTheme}>
  <RadioBox value="light">Claro</RadioBox>
  <RadioBox value="dark">Escuro</RadioBox>
</RadioGroup>;
```

## Propriedades do RadioGroup

**`name`** - `string` (obrigatório)
Nome do campo para manipulação de formulários e agrupamento dos radio buttons

**`label`** - `string`
Texto do rótulo a ser exibido acima do grupo de radio

**`showAsterisk`** - `boolean`
Se deve mostrar asterisco no rótulo para campos obrigatórios

**`errorMessage`** - `string`
Mensagem de erro a ser exibida abaixo do grupo

**`value`** - `string`
Valor controlado para a opção selecionada

**`defaultValue`** - `string` (padrão: `""`)
Valor padrão selecionado para uso não controlado

**`onChange`** - `(value: string) => void`
Função callback chamada quando a seleção muda

**`size`** - `"sm" | "md" | "lg"` (padrão: `"md"`)
Variante de tamanho para todos os radio buttons do grupo

**`className`** - `string`
Classes CSS adicionais para estilização

## Propriedades do RadioBox

**`value`** - `string` (obrigatório)
Valor único para esta opção de radio dentro do grupo

**`isError`** - `boolean`
Indicador de estado de erro para estilização

**`size`** - `"sm" | "md" | "lg"`
Sobrescreve o tamanho herdado do RadioGroup

**`disabled`** - `boolean`
Se esta opção específica está desabilitada

**`children`** - `ReactNode`
Conteúdo a ser exibido ao lado do botão de radio

**`className`** - `string`
Classes CSS adicionais para estilização

## Acessibilidade

O componente `Radio` foi desenvolvido com acessibilidade em mente:

- Uso de `<label>` para associação automática entre radio e conteúdo
- Suporte completo a navegação por teclado (setas para navegar entre opções)
- Estados visuais claros para focus, checked, error e disabled
- Mensagens de erro são anunciadas por leitores de tela
- Gerenciamento adequado de ARIA states
- Agrupamento semântico correto para leitores de tela

## Integração com Formulários

O componente se integra nativamente com sistemas de gerenciamento de formulário:

- Campo hidden automático para envio do valor selecionado
- Validação automática com exibição de erros
- Gerenciamento de estado do formulário
- Suporte ao hook `useForm` do Arkyn

## Notas Importantes

- A propriedade `name` é obrigatória no `RadioGroup` para integração com formulários
- A propriedade `value` é obrigatória em cada `RadioBox` e deve ser única dentro do grupo
- O `RadioBox` deve sempre ser usado como filho direto ou descendente do `RadioGroup`
- O componente gerencia automaticamente o estado de seleção entre as opções
- Apenas uma opção pode estar selecionada por vez dentro de um grupo
- O tamanho definido no `RadioGroup` é herdado pelos `RadioBox`, mas pode ser sobrescrito individualmente
- Estados de erro são propagados automaticamente do grupo para as opções individuais
- O componente usa um input hidden para garantir que o valor seja enviado com o formulário
- Click e focus events são tratados automaticamente para melhor usabilidade
