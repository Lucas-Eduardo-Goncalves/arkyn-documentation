# Textarea

import {
  Textarea,
  AlertContainer,
  AlertContent,
  AlertDescription,
  AlertIcon,
  AlertTitle,
} from "@arkyn/components";

O componente `Textarea` é usado para entrada de texto de múltiplas linhas com estilização personalizável e suporte a validação. É ideal para campos que requerem texto longo como descrições, comentários, mensagens e biografias.

## Importação

```tsx
import { Textarea } from "@arkyn/components";
```

## Propriedades

**`name`** - `string` (obrigatório)
Nome do campo para manipulação de formulários e identificação

**`label`** - `string`
Texto do rótulo a ser exibido acima do textarea

**`showAsterisk`** - `boolean`
Se deve mostrar asterisco no rótulo para campos obrigatórios

**`errorMessage`** - `string`
Mensagem de erro a ser exibida abaixo do textarea

**`size`** - `"md" | "lg"` (padrão: `"md"`)
Variante de tamanho do textarea

**`variant`** - `"solid" | "outline"` (padrão: `"solid"`)
Variante visual do textarea

Todas as outras propriedades HTML padrão de `<textarea>` também são suportadas, exceto `name` que é controlado pelo componente.

## Textarea Controlado

Use `value` e `onChange` para controlar o estado do textarea:

```tsx
const [message, setMessage] = useState("");

<Textarea
  name="controlled-message"
  label="Mensagem Controlada"
  value={message}
  onChange={(e) => setMessage(e.target.value)}
  placeholder="Digite sua mensagem..."
  rows={4}
  showAsterisk
/>;
```

## Acessibilidade

O componente `Textarea` foi desenvolvido com acessibilidade em mente:

- Associação automática entre label e textarea via `htmlFor` e `id`
- Suporte completo a navegação por teclado
- Estados visuais claros para focus, error e disabled
- Mensagens de erro são anunciadas por leitores de tela
- Suporte a `aria-label` e outros atributos ARIA
- Redimensionamento acessível via mouse e teclado

## Integração com Formulários

O componente se integra nativamente com sistemas de gerenciamento de formulário:

- Validação automática considerando a máscara aplicada
- Exibição de erros contextuais
- Gerenciamento de estado do formulário
- Suporte a valores separados ou com máscara

## Notas Importantes

- A propriedade `name` é obrigatória para integração com formulários
- Use `rows` e `cols` para definir o tamanho inicial do textarea
- A propriedade `resize` no style controla como o usuário pode redimensionar
- `maxLength` limita a quantidade de caracteres que podem ser inseridos
- O componente suporta todos os eventos padrão de textarea (onChange, onFocus, onBlur, etc.)
- Estados de disabled e readOnly afetam tanto a aparência quanto a funcionalidade
- O componente herda todas as propriedades HTML padrão de `<textarea>`, exceto `name`
- Para textarea's grandes, considere usar `resize: 'vertical'` para melhor UX
- Mensagens de erro são exibidas abaixo do textarea de forma consistente
- O foco é gerenciado automaticamente quando o usuário clica na área do componente
