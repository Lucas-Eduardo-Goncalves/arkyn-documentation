# Tooltip

import {
  AlertContainer,
  AlertContent,
  AlertDescription,
  AlertIcon,
  AlertTitle,
  Badge,
  Tooltip,
} from "@arkyn/components";

Tooltip displays contextual information when users hover over an element, with automatic position adjustment when space is limited.

```tsx
import { Tooltip } from "@arkyn/components";
```

## Basic Usage

<div className="noContent">
  <Tooltip text="This is a helpful tip">
    <Badge>Hover me</Badge>
  </Tooltip>
</div>

```tsx
<Tooltip text="This is a helpful tip">
    <Badge>Hover me</Badge>
</Tooltip>
```

</Tooltip>

<Tooltip text="Bottom tooltip" orientation="bottom">
  <Badge>Bottom</Badge>

  <Tooltip text="Left tooltip" orientation="left">

  </Tooltip>
</div>

## Smart Positioning

When the preferred orientation does not fit in the viewport, the component automatically flips to a better side.

<div className="noContent">
  <Tooltip
    text="This tooltip automatically adjusts its orientation when needed"
    orientation="left"
  >
    <Badge>Near edge trigger</Badge>
  </Tooltip>
</div>

```tsx
<Tooltip
    text="This tooltip automatically adjusts its orientation when needed"
    orientation="left"
>
    <Badge>Near edge trigger</Badge>
</Tooltip>
```

## Properties

**`text`** - `string`
Text or HTML string displayed inside the tooltip.

**`children`** - `ReactNode`
Element that triggers tooltip visibility on hover.

**`orientation`** - `"top" | "right" | "bottom" | "left"` (default: `"top"`)
Preferred tooltip position relative to the trigger element.

**`size`** - `"md" | "lg"` (default: `"lg"`)
Controls tooltip visual size.

**`className`** - `string` (default: `""`)
Additional CSS classes merged with the component base classes.

**`...rest`** - `Omit<HTMLAttributes<HTMLDivElement>, "children">`
Any other valid HTML attributes for the root `div`, such as `id`, `style`, `aria-*`, and event handlers.
