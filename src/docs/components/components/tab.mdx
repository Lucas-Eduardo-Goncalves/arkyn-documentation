# Tab

import { TabContainer } from "@arkyn/components/tabContainer";
import { TabButton } from "@arkyn/components/tabButton";
import { AlertContainer } from "@arkyn/components/alertContainer";
import { AlertContent } from "@arkyn/components/alertContent";
import { AlertDescription } from "@arkyn/components/alertDescription";
import { AlertIcon } from "@arkyn/components/alertIcon";
import { AlertTitle } from "@arkyn/components/alertTitle";

The Tab family provides accessible tab-like navigation. `TabContainer` holds shared tab state and context, while `TabButton` is the interactive trigger consumed inside it.

```tsx
import { TabContainer } from "@arkyn/components/tabContainer";
import { TabButton } from "@arkyn/components/tabButton";
```

Learn how subpath and root imports differ in [How do I use imports](/docs/guides/how-do-i-use-imports).

## Basic Usage

<div className="noContent column">
  <TabContainer>
    <TabButton value="home">Home</TabButton>
    <TabButton value="about">About</TabButton>
    <TabButton value="contact">Contact</TabButton>
  </TabContainer>
</div>

```tsx
<TabContainer>
    <TabButton value="home">Home</TabButton>
    <TabButton value="about">About</TabButton>
    <TabButton value="contact">Contact</TabButton>
</TabContainer>
```

## Default Value And Change Handler

Use `defaultValue` to initialize the active tab and `onChange` to react to updates.

<div className="noContent column">
  <TabContainer defaultValue="dashboard">
    <TabButton value="dashboard">Dashboard</TabButton>
    <TabButton value="analytics">Analytics</TabButton>
    <TabButton value="settings">Settings</TabButton>
  </TabContainer>
</div>

```tsx
<TabContainer
    defaultValue="dashboard"
    onChange={(value) => console.log("Active tab:", value)}
>
    <TabButton value="dashboard">Dashboard</TabButton>
    <TabButton value="analytics">Analytics</TabButton>
    <TabButton value="settings">Settings</TabButton>
</TabContainer>
```

## Click Handling

`TabButton` supports custom click handlers while still updating tab context.

<div className="noContent column">
  <TabContainer>
    <TabButton value="profile">Profile</TabButton>
    <TabButton value="settings">Settings</TabButton>
  </TabContainer>
</div>

```tsx
<TabContainer onChange={(value) => console.log("Tab changed:", value)}>
    <TabButton value="profile" onClick={() => console.log("Profile tab clicked")}>
        Profile
    </TabButton>
    <TabButton value="settings">Settings</TabButton>
</TabContainer>
```

## Disabled

Set `disabled` on `TabContainer` to disable all child tab buttons, or on an individual `TabButton` to disable just that tab.

<div className="noContent column">
  <TabContainer disabled>
    <TabButton value="tab1">Tab 1</TabButton>
    <TabButton value="tab2">Tab 2</TabButton>
  </TabContainer>

  <TabContainer>
    <TabButton value="available">Available</TabButton>
    <TabButton value="coming-soon" disabled>
      Coming Soon
    </TabButton>
  </TabContainer>
</div>

```tsx
<TabContainer disabled>
    <TabButton value="tab1">Tab 1</TabButton>
    <TabButton value="tab2">Tab 2</TabButton>
</TabContainer>

<TabContainer>
    <TabButton value="available">Available</TabButton>
    <TabButton value="coming-soon" disabled>
        Coming Soon
    </TabButton>
</TabContainer>
```

## Integration With Page State

Use external state when tab selection must drive conditional content rendering.

<div className="noContent column">
  <TabContainer defaultValue="profile" className="custom-tabs" role="tablist">
    <TabButton value="profile">Profile</TabButton>
    <TabButton value="account">Account</TabButton>
    <TabButton value="notifications">Notifications</TabButton>
  </TabContainer>
</div>

```tsx
function MyTabs() {
    const [activeTab, setActiveTab] = useState("profile");

    return (
        <div>
            <TabContainer
                defaultValue={activeTab}
                onChange={setActiveTab}
                className="custom-tabs"
                role="tablist"
            >
                <TabButton value="profile">Profile</TabButton>
                <TabButton value="account">Account</TabButton>
                <TabButton value="notifications">Notifications</TabButton>
            </TabContainer>

            {activeTab === "profile" && <ProfileContent />}
            {activeTab === "account" && <AccountContent />}
            {activeTab === "notifications" && <NotificationsContent />}
        </div>
    );
}
```

<AlertContainer schema="info">
  <AlertIcon />
  <AlertContent>
    <AlertTitle>Context provider behavior</AlertTitle>
    <AlertDescription>
      `TabContainer` provides tab state through context. Child `TabButton`
      components consume this context to determine active/disabled behavior.
    </AlertDescription>
  </AlertContent>
</AlertContainer>

## Properties

### TabContainer

**`children`** - `ReactNode`
Tab buttons and related content rendered inside the navigation container.

**`disabled`** - `boolean` (default: `false`)
Disables all tabs within the container.

**`defaultValue`** - `string`
Initial active tab value.

**`onChange`** - `(index: string) => void`
Called when active tab changes.

**`className`** - `string`
Additional classes for the root `nav` element.

**`role`** - `AriaRole`
ARIA role applied to the root `nav` element.

**`id`** - `string`
Optional id for the root `nav` element.

**`style`** - `CSSProperties`
Inline styles applied to the root `nav` element.

**`...rest`** - `Omit<HTMLAttributes<HTMLElement>, "onChange" | "children" | "ref" | "onClick">`
Any other valid native `nav` attributes are forwarded except excluded ones.

### TabButton

**`children`** - `ReactNode`
Content displayed inside the tab button.

**`value`** - `string`
Unique identifier used to activate and track the current tab.

**`disabled`** - `boolean`
Disables this specific tab button.

**`className`** - `string`
Additional classes for the button element.

**`onClick`** - `(event: MouseEvent<HTMLButtonElement>) => void`
Custom click handler called alongside internal tab selection.

**`aria-label`** - `string`
Optional accessibility label.

**`title`** - `string`
Native title attribute.

**`id`** - `string`
Optional id for the button element.

**`...rest`** - `Omit<ButtonHTMLAttributes<HTMLButtonElement>, "children" | "value" | "type">`
Any other valid button attributes are forwarded except excluded ones.
