# FullCalendar

import {
  FullCalendar,
  AlertContainer,
  AlertContent,
  AlertDescription,
  AlertIcon,
  AlertTitle,
} from "@arkyn/components";

Full-featured calendar component with switchable day, week, and month views, supporting event display, blocked time ranges, and date navigation.

```tsx
import { FullCalendar } from "@arkyn/components";
```

## Basic Usage

<div className="noContent" style={{ minWidth: "790px" }}>
  <FullCalendar defaultValue={new Date()} />
</div>

```tsx
<FullCalendar defaultValue={new Date()} />
```

## Events

Pass an array of event objects to the `events` prop. Each event requires a `title` and an `initialDate`. Events are rendered as colored chips on the corresponding day or time slot across all three views.

    const yesterday = new Date();

    yesterday.setHours(8, 0, 0, 0);

    const nextWeek = new Date();
    nextWeek.setDate(nextWeek.getDate() + 7);

    return (
      <FullCalendar
        defaultValue={new Date()}
        events={[
          {

            initialDate: today,
            endDate: new Date(today.getTime() + 30 * 60 * 1000),
            scheme: "primary",
          },
          {
            title: "Doctor appointment",
            initialDate: new Date(today.getTime() + 90 * 60 * 1000),
            endDate: new Date(today.getTime() + 150 * 60 * 1000),
            scheme: "success",
          },
          {
            title: "Lunch with Maria",
            initialDate: dayAfterTomorrow,
            endDate: new Date(dayAfterTomorrow.getTime() + 60 * 60 * 1000),
            scheme: "info",
          },
          {
            title: "Dentist",
            initialDate: yesterday,
            endDate: new Date(yesterday.getTime() + 60 * 60 * 1000),
            scheme: "danger",
          },

            initialDate: nextWeek,
            endDate: new Date(nextWeek.getTime() + 90 * 60 * 1000),

  })()}
</div>

## Event Color Schemes

Use the `scheme` property on each event to apply a color: `"primary"` (blue), `"success"` (green), `"warning"` (yellow), `"danger"` (red), and `"info"` (light blue). Defaults to `"primary"` when omitted.

```tsx
{ title: "Meeting",     initialDate: date, scheme: "primary" }
{ title: "Confirmed",   initialDate: date, scheme: "success" }
{ title: "Pending",     initialDate: date, scheme: "warning" }
{ title: "Cancelled",   initialDate: date, scheme: "danger"  }
{ title: "Note",        initialDate: date, scheme: "info"    }
```

## Event Click Handler

Attach an `onClick` callback to individual events to respond to user interaction. The callback receives the `data` payload defined on that event.

```tsx
<FullCalendar
  events={[
    {
      title: "Team meeting",
      initialDate: new Date(2026, 5, 22, 10, 0),
      endDate: new Date(2026, 5, 22, 11, 0),
      scheme: "primary",
      data: { meetingId: 42 },
      onClick: ({ meetingId }) => openMeeting(meetingId),
    },
  ]}
/>
```

## Blocked Timestamps

Use `blockedTimestamps` to visually mark time ranges as unavailable. Each entry requires an `initialDate` and an `endDate`. Blocked ranges are rendered differently from events and are not interactive.

```tsx
<FullCalendar
  defaultValue={new Date()}
  blockedTimestamps={[
    {
      initialDate: new Date(2026, 5, 22, 12, 0),
      endDate: new Date(2026, 5, 22, 13, 0),
    },
    {
      initialDate: new Date(2026, 5, 23, 8, 0),
      endDate: new Date(2026, 5, 23, 9, 0),
    },
  ]}
/>
```

## Navigation

Use `onChangeView` to react when the user navigates to a different period (e.g., next month or previous week). The callback receives the `Date` representing the newly visible period — useful for fetching events on demand.

Use `onClickDate` to respond when the user clicks a date cell.

```tsx
<FullCalendar
  onChangeView={(date) => fetchEventsForPeriod(date)}
  onClickDate={(date) => openCreateEventModal(date)}
/>
```

## Properties

**`defaultValue`** - `Date`
Initial focused date when the calendar mounts. Defaults to today when not provided.

**`events`** - `FullCalendarEvent[]`
Array of events to display across day, week, and month views.

**`blockedTimestamps`** - `BlockTimestamp[]`
Time ranges that should be visually marked as unavailable on the calendar.

**`onChangeView`** - `(date: Date) => void`
Callback fired when the user navigates to a different period. Receives the `Date` of the newly visible period.

**`onClickDate`** - `(date: Date) => void`
Callback fired when the user clicks a date cell. Receives the clicked `Date`.

---

### Event Properties (`FullCalendarEvent`)

**`title`** - `string`
Display label shown on the event chip. Required.

**`initialDate`** - `Date`
Start date and time of the event. Required.

**`endDate`** - `Date`
End date and time of the event. When omitted, the event is treated as a point in time.

**`data`** - `any`
Arbitrary payload forwarded to the `onClick` callback.

**`scheme`** - `"primary" | "success" | "warning" | "danger" | "info"` (default: `"primary"`)
Color scheme applied to the event chip.

**`onClick`** - `(data: any) => void`
Callback fired when the user clicks the event chip. Receives the value of `data`.

---

### Blocked Timestamp Properties (`BlockTimestamp`)

**`initialDate`** - `Date`
Start of the blocked time range. Required.

**`endDate`** - `Date`
End of the blocked time range. Required.
