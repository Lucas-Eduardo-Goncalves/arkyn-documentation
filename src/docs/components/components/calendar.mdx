# Calendar

import { AlertContainer } from "@arkyn/components/alertContainer";
import { AlertContent } from "@arkyn/components/alertContent";
import { AlertDescription } from "@arkyn/components/alertDescription";
import { AlertIcon } from "@arkyn/components/alertIcon";
import { AlertTitle } from "@arkyn/components/alertTitle";
import { Calendar } from "@arkyn/components/calendar";

The `Calendar` component provides single-date and date-range selection with built-in month navigation for scheduling, filtering, and booking flows.

```tsx
import { Calendar } from "@arkyn/components/calendar";
```

Learn how subpath and root imports differ in [How do I use imports](/docs/guides/how-do-i-use-imports).

## Basic Usage

Use `type="single"` to select one date.

<div className="noContent">
	<Calendar type="single" />
</div>

```tsx
<Calendar type="single" />
```

## Selection Modes

`Calendar` supports two selection modes: `single` for one date and `range` for a start/end interval.

<div className="noContent column">
	<Calendar type="single" defaultValue={new Date(2026, 5, 10)} />
	<Calendar
		type="range"
		defaultValue={[new Date(2026, 5, 1), new Date(2026, 5, 15)]}
	/>
</div>

```tsx
<Calendar 
  type="single" defaultValue={new Date(2026, 5, 10)} 
/>

<Calendar
	type="range"
	defaultValue={[new Date(2026, 5, 1), new Date(2026, 5, 15)]}
/>
```

## Variants

Use `variant` to switch between simplified and full header navigation.

<div className="noContent column">
	<Calendar type="single" variant="basic" />
	<Calendar type="single" variant="complete" />
</div>

```tsx
<Calendar type="single" variant="basic" />
<Calendar type="single" variant="complete" />
```

## Controlled Mode

Use `value` to control the selected date externally (controlled component pattern). Pair with `onChange` to keep state in sync.

```tsx
const [date, setDate] = useState<Date | undefined>(undefined);

<Calendar
	type="single"
	value={date}
	onChange={(d) => setDate(d)}
/>
```

For range mode:

```tsx
const [range, setRange] = useState<[Date, Date] | undefined>(undefined);

<Calendar
	type="range"
	value={range}
	onChange={(r) => setRange(r)}
/>
```

## Controlled View

Use `viewValue` to control the currently visible month/year, and `defaultViewValue` to set the initial view without making it fully controlled.

```tsx
<Calendar
	type="single"
	defaultViewValue={new Date(2026, 0, 1)}
	onChangeView={(viewDate) => {
		console.log("Navigated to:", viewDate);
	}}
/>
```

## Change Callbacks

Use `onChange` to react to date selection changes and `onChangeView` to react when the visible month/year changes.

<div className="noContent column">
	<Calendar
		type="single"
		onChange={(date) => {
			console.log("Selected date:", date);
		}}
		onChangeView={(viewDate) => {
			console.log("Current view:", viewDate);
		}}
	/>

	<Calendar
		type="range"
		onChange={([start, end]) => {
			console.log("Selected range:", start, end);
		}}
		onChangeView={(viewDate) => {
			console.log("Focused month:", viewDate);
		}}
	/>
</div>

```tsx
<Calendar
	type="single"
	onChange={(date) => {
		console.log("Selected date:", date);
	}}
	onChangeView={(viewDate) => {
		console.log("Current view:", viewDate);
	}}
/>

<Calendar
	type="range"
	onChange={([start, end]) => {
		console.log("Selected range:", start, end);
	}}
	onChangeView={(viewDate) => {
		console.log("Focused month:", viewDate);
	}}
/>
```

## Properties

**`type`** - `"single" | "range"`
Defines the calendar selection mode.

**`variant`** - `"basic" | "complete"` (default: `"complete"`)
Controls header behavior and navigation complexity.

**`value`** - `Date` (when `type="single"`)
Controlled selected date in single mode.

**`value`** - `[Date, Date]` (when `type="range"`)
Controlled selected range in range mode, using `[start, end]` format.

**`defaultValue`** - `Date` (when `type="single"`)
Initial selected date in single mode (uncontrolled).

**`defaultValue`** - `[Date, Date]` (when `type="range"`)
Initial selected range in range mode (uncontrolled), using `[start, end]` format.

**`onChange`** - `(date: Date) => void` (when `type="single"`)
Called when the selected single date changes.

**`onChange`** - `(date: [Date, Date]) => void` (when `type="range"`)
Called when the selected date range changes.

**`viewValue`** - `Date`
Controlled currently visible month/year.

**`defaultViewValue`** - `Date`
Initial visible month/year (uncontrolled).

**`onChangeView`** - `(date: Date) => void`
Called when the current calendar view changes (month/year navigation).
